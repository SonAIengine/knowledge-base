# 업무 일일 리포트 생성

{DATE} 하루 동안의 모든 업무 커뮤니케이션(Teams 채팅, Teams 채널, 메일)을 분석하여 **이 리포트 하나만 읽으면 하루 전체를 완벽히 파악할 수 있는** 종합 업무 리포트를 생성합니다.

**본인 이름**: {MY_NAME} (액션 아이템에서 "내가 처리할 건" vs "다른 사람에게 요청한 건" 구분 기준)

## 핵심 원칙

1. **구체성**: "논의함" 대신 "A가 B에게 ~를 요청, C 방식으로 결정" 수준으로 작성
2. **맥락 보존**: 나중에 이 리포트를 다시 봤을 때 "이게 무슨 건이었지?"가 안 되도록, 배경/이유를 반드시 포함
3. **핵심 발언 인용**: 중요한 의사결정, 지시, 수치가 포함된 발언은 `> 인용` 형식으로 원문 보존
4. **추적 가능성**: 누가 → 누구에게 → 무엇을 → 왜 → 언제까지 의 5W 구조

## 제외 대상
- 인사말, 단순 확인 ("네", "넵", "알겠습니다", "수고하세요")
- 빈 메시지 (content가 비어있는 것)
- 시스템 자동 메일 (noreply@, Microsoft Teams 알림, bot 발신)
- 전사 공지 메일 (HR 근태마감 등) — 행동이 필요한 경우만 포함
- 이모지/스티커만 있는 메시지
- 잡담, 점심 메뉴, 날씨 등 업무 무관 대화

## 포함 대상
- 업무 요청/지시 (누가 누구에게, 무엇을, 왜)
- 완료 보고 (배포, 계정 신청, 버그 수정 등)
- 기술 토론 (에러 분석, 아키텍처 결정, 코드 리뷰)
- 외부 커뮤니케이션 (고객사/파트너)
- 회의/일정 관련
- 공유된 기술 자료/문서 링크
- 수치/금액/사양 등 구체적 정보

## 첨부파일
- 메일 메시지에 `attachments` 배열이 있으면 해당 섹션에서 마크다운 링크로 표시
- 링크 형식: `[파일명](daily/attachments/{DATE}/파일명 ':ignore')`
  - 경로는 반드시 `daily/attachments/`로 시작할 것 (Docsify 루트 기준)
  - 반드시 `':ignore'`를 붙일 것 (Docsify에서 직접 다운로드되도록)
- 프로젝트별 현황, 외부 대응 등 관련 섹션에 자연스럽게 포함
- 별도 "첨부파일" 섹션은 만들지 않음 — 해당 맥락에 인라인으로 삽입

## 핵심 요약
- 오늘의 가장 중요한 3~5가지 사항 (bullet point)
- 각 항목에 **구체적 결과와 영향**을 포함
- 단순 "논의함"이 아니라 "A를 B로 결정, C까지 반영 예정" 수준으로 작성
- 하루를 한눈에 파악할 수 있는 executive summary

## 오늘의 의사결정
- 오늘 확정/합의된 사항만 기록 (아직 논의 중인 건 제외)
- 형식: **[프로젝트/주제]** 결정 내용 — 배경/이유 (결정자)
- 예: **[XGEN]** 문서관리 조회를 qdrant → PostgreSQL로 전환 — 3초→0.1초 성능 개선 목적 (유지수 제안, 손성준 승인)

## 액션 아이템

### 내가 처리할 건
| 내용 | 배경/맥락 | 요청자 | 출처 | 기한 |
|------|-----------|--------|------|------|
| 구체적으로 뭘 해야 하는지 | 왜 필요한지, 어떤 상황에서 나온 건지 | 누가 요청 | 채널 | 언제까지 |

### 다른 사람에게 요청한 건
| 내용 | 배경/맥락 | 담당자 | 출처 | 기한 | 현재 상태 |
|------|-----------|--------|------|------|-----------|
| 구체적으로 뭘 요청했는지 | 왜 필요한지 | 누구에게 | 채널 | 언제까지 | 대기중/진행중/회신완료 |

## 프로젝트별 현황

### {프로젝트명}

**오늘 진행된 일**:
- 구체적으로 오늘 완료되거나 진행된 작업 (담당자 명시)
- 중요한 대화는 핵심 발언을 인용:
  > "원문에서 중요한 부분" — 발언자

**기술 상세**:
- 에러 원인, 해결 방법, 설정값, 코드 경로 등 나중에 참조할 기술 정보
- 아키텍처 결정과 그 이유

**이슈/리스크**:
- 해결 필요한 문제와 현재 상태
- 누가 해결 중인지, 언제 해결 예정인지

**다음 단계**:
- 이 프로젝트에서 다음에 해야 할 일 (내일/차주)

(프로젝트명은 chatTopic, 메일 subject, 대화 내용에서 자동 식별)

## 규칙
- **원문의 핵심 발언은 `>` 인용으로 보존** — 특히 의사결정, 지시, 수치가 포함된 발언
- 참여자 이름은 원문 그대로 사용
- 한국어로 작성
- front-matter의 teams_chat/teams_channel/mail 숫자는 입력 데이터의 stats에서 메시지 수를 가져올 것
- 프로젝트별 현황의 프로젝트명은 대화 내용 기반으로 자동 식별 (chatTopic, subject 등 활용)
- **모호한 표현 금지**: "관련 논의", "검토 예정" 같은 표현 대신 구체적으로 누가/무엇을/언제 하는지 명시
- 같은 정보가 여러 섹션에 중복되지 않도록 주의 (프로젝트 현황에 상세 기술 → 기술 메모에서는 참조만)
- 액션 아이템의 "배경/맥락" 컬럼은 간결하게 1줄로 (상세 내용은 프로젝트별 현황에서 다룸)
